## 7.1 Rollout plan model
Implement `src/core/rollout.ts`:

Define a rollout plan that caps writes per day and supports gradual ramp-up:
- Day 1: 200 writes
- Day 2: 1,000 writes
- Day 3: 3,000 writes
- Day 4+: 10,000 writes

Make it configurable:
- env or CLI can override the schedule
- allow a custom plan JSON file

Rollout plan structure:
```ts
export type RolloutPlan = {
  version: 1
  timezone: "Asia/Singapore"
  stages: Array<{
    dayIndex: number // 1-based
    maxWrites: number
    note?: string
  }>
  steadyStateMaxWrites: number
}
```

Add helper:
* getMaxWritesForDate(plan, dateISO) (based on first activation date)
* loadRolloutPlan(filePath?) (default built-in)
* writeRolloutPlanPreview(outDir) (writes json + md)
